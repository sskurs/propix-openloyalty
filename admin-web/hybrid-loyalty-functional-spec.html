1. System Overview

The Hybrid Loyalty System allows customers (members) to earn and redeem points, progress through tier levels, receive cashback/store credit, use coupons & vouchers, participate in campaigns, and unlock achievements/gamification rewards.

The system supports:

  • Online & offline channels (POS + eCommerce + Mobile)
  • Multiple wallets (Points, Cashback, Gift, Promotional)
  • Multiple currencies and languages where required
  • Full member timeline tracking similar to Open Loyalty’s timeline

2. Core Business Entities

Entity Description
Member Registered customer participating in the loyalty program.
Points Wallet Balance of points earned from transactions and campaigns; subject to expiry rules.
Cashback Wallet Monetary-equivalent balance (store credit) generated via cashback rules.
Gift / Promo Wallet Special wallet for gift points, promo credits, or one-time grants.
Tiers Loyalty levels (e.g., Bronze, Silver, Gold, Platinum) with progressive benefits.
Campaigns Configurable rules that award points, cashback, or rewards based on events.
Rewards Catalog items, vouchers, coupons, or gifts redeemable using points or cashback.
Coupons Codes applied at checkout/POS to receive discounts or free items.
Achievements Gamified milestones: badges, missions, streaks, or challenges.
Transactions All monetary purchases, redemptions, adjustments, refunds, and expiries.
Timeline Events Unified chronological log of all important member-related events (similar to Open Loyalty Timeline).
Store / Channel Where the activity occurred (online store, physical branch, mobile app, partner, etc.).

3. Hybrid Loyalty Rules

Feature Functional Rules
Points Earning
  • Earn rate defined as x points / currency unit per channel, category, or product.
  • Tier multipliers (e.g., Silver: 1.2×, Gold: 1.5× base points).
  • Special campaigns (double points, category-based bonuses, birthday multipliers).
  • Support for negative/zero earn events (e.g., excluded products, staff purchases).
Redemption Rules
  • Redeem points for: inline discount, vouchers, catalog rewards, or cashback.
  • Minimum balance and minimum redemption thresholds.
  • Channel-based restrictions (e.g., online only, specific stores only).
  • Conversion rate configuration (e.g., 100 points = ₹10).
Cashback Generation
  • Campaign-based cashback: percentage of order value or fixed amount.
  • Brand/category/store specific cashback rules.
  • Cashback expiry policy independent from points.
Tier Progression
  • Progress based on yearly spend, number of transactions, or tier points.
  • Auto-upgrade when threshold is met; optional scheduled job to evaluate tiers.
  • Downgrade policy (e.g., after 12 months of inactivity or threshold not met).
Expiry Policy
  • First Earn First Expire logic for points.
  • Global or wallet-specific expiry duration (e.g., 365 days).
  • Expiry jobs running daily and logged as timeline events.
Returns / Refunds
  • Reverse previously awarded points/cashback for refunded orders.
  • Reverse redemptions or adjust balances when orders are cancelled.
Offline / POS
  • Support for delayed synchronization of transactions.
  • Conflict/resync policies when the same member performs actions offline and online.
Fraud Prevention
  • Daily/weekly caps on points earning and redemption.
  • Manual review flags for suspicious activity.
  • IP/device tracking (if applicable) and blacklisting.

4. Admin Portal – Menu & Submenus

4.1 Dashboard

  • KPIs: Total members, active members, newly joined, total points issued, total points redeemed, points breakage.
  • Charts: Daily/weekly points issuance vs redemption, campaign performance, tier distribution.
  • Recent Activity: Latest member registrations, large redemptions, suspicious events.

4.2 Members

  • Member List
    • Search and filter by: name, ID, phone, email, tier, join date, status.
    • Bulk actions: export, tag, activate/deactivate.
  • Member Profile View (Single Member View)
    • Profile & Attributes
    • Wallet Balances (with drill-down)
    • Transactions (earn, redeem, expiry, adjustments)
    • Timeline – chronological events:
      • Points earned / redeemed / expired / adjusted
      • Tier changes (upgrade/downgrade)
      • Campaign participation events
      • Achievements unlocked
      • Coupons issued / used
      • Member data changes (optional)
    • Campaigns enrolled & performance
    • Vouchers & Coupons assigned
    • Communication log (emails/SMS/push sent)
    • Internal notes & tags

4.3 Wallets

  • Wallet Types configuration (Points, Cashback, Gift, etc.).
  • Wallet Transaction Logs with filters and export.
  • Manual Wallet Adjustment:
    • Credit/debit points or cashback with mandatory reason.
    • Audit trail for each manual adjustment.
  • Expiry Rules & Scheduler:
    • Global expiry settings per wallet.
    • Simulation of upcoming expiries.

4.4 Tiers

  • Tier Definition (name, color, badge icon, sort order).
  • Progression Rules:
    • Spend thresholds, number of transactions, or points.
    • Tracking period (rolling 12 months, calendar year, lifetime).
  • Tier Benefits configuration:
    • Points multiplier.
    • Exclusive campaigns eligibility.
    • Free shipping, special discounts, priority customer service.
  • Tier Downgrade rules.
  • Tier History (timeline of tier changes per member).

4.5 Campaigns

  • Campaign List (status, period, KPIs).
  • Create / Edit Campaign including:
    • Basic details: name, description, validity dates.
    • Target segment: tiers, demographics, RFM score, custom segments.
    • Conditions (IF section): event type, order value, categories, channels.
    • Rewards (THEN section): points, cashback, vouchers, free gifts.
    • Limits: per member, per day, global cap.
  • Rule Builder UI:
    • Nested conditions with AND/OR logic.
    • Preview / test mode with sample transaction.
  • Campaign Reports:
    • Number of participants, points issued, revenue uplift.
    • Conversion rates per channel.

4.6 Rewards

  • Reward Catalog management:
    • Physical products, digital vouchers, coupons, experiences.
    • Fields: name, image, description, category, stock, cost in points/cashback.
  • Restrictions:
    • Min/max tier, channel, store.
    • Per-member redemption limits.
  • Redemption history and stock tracking.

4.7 Achievements & Gamification

  • Badge configuration (name, icon, criteria).
  • Missions/Challenges (e.g., “place 5 orders in a month”).
  • Streaks (consecutive days/weeks of activity).
  • Rewards for completing achievements (points, badges, vouchers).
  • Achievement logs and member-level views.

4.8 Coupons

  • Coupon generation:
    • Static codes (e.g., WELCOME10).
    • Dynamic/generated codes per member.
  • Redemption rules:
    • Discount type: fixed amount, percentage, free item.
    • Minimum basket size, product/category restrictions.
    • Validity window, max uses per member, global caps.
  • Redemption history and performance metrics.

4.9 Communication

  • Channels: Email, SMS, Push notifications, optional WhatsApp.
  • Template management with personalization variables (name, tier, points, etc.).
  • Triggers:
    • Welcome, birthday, near-expiry, tier upgrade, campaign invite, post-purchase.
  • Communication logs per member and per campaign.

4.10 Reports & Analytics

  • Member lifecycle analysis (new vs. active vs. dormant).
  • Tier migration reports.
  • Points breakage (expired vs. redeemed).
  • Campaign ROI and attribution.
  • Fraud and anomaly detection report.

4.11 System Settings

  • Attribute management (custom fields on members, transactions, events).
  • Store/branch/channel setup.
  • Taxonomy: categories, brands, tags.
  • Scheduled jobs:
    • Points expiry, tier evaluation, campaign housekeeping.
  • Integration keys for POS, eCommerce, ERP, CRM.
  • Role-based access control (RBAC) and permissions.

5. Member Portal – Menu Structure

5.1 Home

  • Current tier badge and name.
  • Progress bar to next tier (e.g., “₹12,000 more to reach Gold”).
  • Points wallet balance and cashback balance.
  • Top campaigns and recommended rewards.

5.2 Rewards

  • Reward catalog with filters (category, cost, channel).
  • Reward details page:
    • Description, image, cost, availability, restrictions.
    • Redeem button, confirmation step, and success screen.
  • My redemptions history.

5.3 My Wallet

  • Overview of all wallets: Points, Cashback, Gift/Promo.
  • Transaction history for each wallet (earn, redeem, expire, adjust).
  • Upcoming expiries with dates.

5.4 My Tiers

  • Current tier and benefits list.
  • Comparison with higher tiers (what extra you get).
  • Progress metrics (spend, points, orders).

5.5 Achievements

  • Badges earned with dates and descriptions.
  • Active missions/challenges and rewards for completion.
  • Streaks view and countdowns.

5.6 Timeline View

A chronological activity feed similar to Open Loyalty’s timeline, showing key events in the member’s journey.

  • Points earned, redeemed, expired.
  • Tier upgrades/downgrades.
  • Rewards redeemed, vouchers issued/used.
  • Campaign participation and achievement unlocks.

5.7 Profile & Settings

  • Personal data (name, contact, DOB, etc.).
  • Communication preferences and consent.
  • Linked accounts (social login, mobile, partner IDs).
  • Data export / delete account (where required by regulation).

7. Technical & Integration Requirements

Interface Requirement
eCommerce REST/GraphQL hooks at checkout to calculate and apply earn/redeem, and to display real-time wallet balances.
POS APIs to identify member, earn/redeem at POS, and queue transactions for sync when offline.
ERP/CRM Order, return, and customer sync; ability to update attributes and segments.
Mobile Push Integration with push providers (e.g., Firebase/APNs) for real-time notifications.
Authentication OAuth2 / JWT-based SSO; optional social login (Google, Facebook, Apple ID).

Public APIs should be versioned, secured with API keys and member auth tokens, and documented via OpenAPI/Swagger.

8. Data Model (High Level)

The following are key logical tables/entities (technology-agnostic):

  • Member (id, external_ids, name, contact, attributes, tier_id, join_date, status)
  • Transaction (id, member_id, type, order_id, channel, value, currency, date)
  • Wallet (id, member_id, type, balance, currency)
  • Wallet_Log (id, wallet_id, delta, reason, transaction_id, expiry_date, created_at)
  • Tier (id, code, name, thresholds, multiplier, benefits)
  • Tier_History (id, member_id, tier_id, from_date, to_date, reason)
  • Reward (id, name, type, stock, cost_points, cost_cashback, restrictions)
  • Coupon (id, code, type, discount, rules, validity, status)
  • Campaign (id, name, rules_json, start_date, end_date, status)
  • Campaign_Log (id, campaign_id, member_id, event_id, reward_value, created_at)
  • Achievement (id, name, criteria_json, reward_json)
  • Achievement_Log (id, member_id, achievement_id, unlocked_at)
  • Timeline_Event (id, member_id, event_type, payload_json, created_at, source_channel)

Timeline_Event is the central log for the member’s journey, aggregating different event types into a single chronological feed, similar to Open Loyalty’s Timeline feature.

9. Security & Compliance

  • Role-based access control (RBAC) for admin portal.
  • Encryption for sensitive fields (e.g., phone, email) at rest and in transit (TLS).
  • Comprehensive audit logs for all admin actions and manual adjustments.
  • Data retention policies and anonymization/deletion workflows.
  • Consent management and communication preference storage.

10. Roadmap Extensions (Optional)

  • Family / household accounts and shared wallets.
  • Multi-brand / coalition loyalty with partner integrations.
  • Referral program with tracking and rewards.
  • Geofencing-based offers and in-store triggers.
  • AI-based reward recommendations and churn prediction.
  • Conversational chatbot interface over WhatsApp or in-app chat.
